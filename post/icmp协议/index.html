ICMP协议

ICMP协议 全称是 Internet Control Message Protocol 是与IP协议同属于第三层网络层的，是一种“错误侦测与回报机制”，其目的是便于我们检测网络的连线状况，其功能有：

  1. 侦测远端主机是否存在。
  2. 建立及维护路由资料。
  3. 重导资料传送路径。
  4. 资料流量控制。
  • ping

其中ping命令就是是用ICMP协议包，使用ping命令可以帮助我们排查诸多状况，例如：

  1. ping不通主机名或域名，ping得通ip，很可能dns不工作了。
  2. ping对方的router（如果您知道其地址的话），假如ping得上，那可能是对方机器和其相连网络的问题；
  3. 如果ping不到对方的router，那么可以ping自己的router。如果ping得上，那么好可能是router和router之间的问题；
  4. 如果自己的router也ping不到，那么可能是自己的机器和router之间的问题，我们可以ping一下自己的IP。如果自己的IP可以ping得到，那么可能是连线的问题，我们可以检查一下网线、hub等设备，看看有没有损坏的状况。
  5. 同时，我们也可以ping一下网络上其他机器，也可以用其它机器ping一下router，来判别一下问题来自自己机器，还是网路，还是router等等。
  6. 如果自己的IP都ping不到，那么可能是网卡坏掉或者没有正确设定，可以看看设备资源有没有冲突，也可以看看设备有没有被系统启动。
  7. 如果看来都没问题，那么可以ping一下回环地址127.0.0.1， 如果连这个都ping不了的话，这台机器的IP功能根本没有被启动。那么，你就要检查一下网络功能有没有选择，IP协定有没有被绑定（bind）等基本网络设定了。

使用ping命令 实际上是送出一个echo-request(type 8)的ICMP包，如果对方机器收到这个请求，并且愿意做出回应，则送回一个echo-reply(type 0)的ICMP包，当这回应顺利到达时，那就完成一个ping的动作。

那么可以设置机器不回应任何echo-request包，若在Linux上，只要用下面命令就可以了：

1

echo "1" > /proc/sys/net/ipv4/icmp_echo_ignore_all

  • traceroute命令（windows tracert命令）

    TTL是 Time To Live的缩写，该字段指定IP包被路由器丢弃之前允许通过的最大网段数量。

    traceroute命令就是利用ICMP包的TTL，本质上是IP包的TTL。看它如何运作的：

    1. 首先，traceroute命令会向目标地址送出UDP侦测封包（在Linux中，可用-I改为ICMP封包），但将第一个送出的封包之TTL设为1，这样，第一个路由节点在处理这个封包的时候，减掉1，并发现TTL为0， 于是就不处理这个封包，并同时送回一个ICMP封包。这样，发送端就知道第一个路由节点在哪里了。
    2. 当接得到第一个ICMP返回的时候，程序会检查返回主机是否就是目标主机，如果不是，则再送出第二个封包，但TTL比上次增加1。
    3. 这样，第一路由节点接到的封包之TTL就不是0，那么处理完毕后送给下一个节点，同时将TTL扣除1。这样，当下一个站收到这个封包，再扣掉TTL为0，也会送回ICMP封包，这样，程序就知道第二个路由节点在哪里了。
    4. 然后重复上述2，3动作，直到找到目标主机为止，或是封包的最大TTL（通常为30）都用光为止，但您可以用-m参数来指定最大的TTL值。

    另外如果traceroute的某行结果维持*星号，那可能是因为ICMP被对方的防火墙拦下来的结果。这样就无法完成防火墙后的路由追踪了。
